# Security Policy

## Supported Versions

WorkOrder is in early development. Security updates will be prioritized for the latest version.

| Version | Supported |
| ------- | --------- |
| Latest  | ✓         |
| Older   | ✗         |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do not** open a public issue for security vulnerabilities.

Instead, please report security issues through GitHub's [security advisory system](https://github.com/workorder-systems/workorder/security/advisories/new).

### What to Include

When reporting a security vulnerability, please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### Response Timeline

We will acknowledge receipt of your report within 48 hours and aim to provide an initial assessment within 7 days. We'll keep you informed of our progress and work with you to address the issue.

### Disclosure Policy

We follow responsible disclosure practices:
- We will notify you when we've received your report
- We'll keep you updated on our progress
- We'll work with you to understand and resolve the issue
- We'll coordinate public disclosure after a fix is available

Thank you for helping keep WorkOrder secure.
